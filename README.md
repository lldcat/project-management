# PM 项目成本、进度及绩效管理小程序

## 一、系统简介

本系统是面向业务人员的微信小程序，用于把 Pre-cal、SAP 项目号、正式项目、AR Time 和项目绩效数据串起来，帮助 Sales、CS、PM / 项目负责人在同一个流程里查看项目预算、工时和执行情况。

主要用于：

- Pre-cal 项目预算信息管理
- SAP 项目号绑定
- 项目创建与项目追踪
- AR Time 与项目人员工时统计
- 成本、进度、绩效数据查看

当前仓库代码只包含微信小程序入口，没有独立的 `ar-pc-web` 或 PC Web 前端。普通用户请从小程序底部导航进入“首页”“项目”“业务”“我的”等页面使用系统。

## 二、普通用户使用流程总览

整体业务流程如下：

1. Sales 在“业务”页进入“新建 Pre-cal”，填写客户、服务明细、订单金额、人天和相关费用。
2. Sales 保存或提交 Pre-cal。提交后，Pre-cal 才会进入 CS 可处理范围。
3. CS 在“业务”页进入“SAP 绑定”，找到已提交的 Pre-cal，并补充 SAP 项目号、Item 号、关联人员和备注。
4. PM / 项目负责人在“业务”页进入“创建项目”，输入任意一个已绑定的 SAP 项目号。
5. 系统自动查找对应 Pre-cal，并带入客户名称、项目预算、预算工时、差旅费、子项目、成员、SAP / Item 信息。
6. 系统根据 SAP 项目号匹配 AR Summary 中的人员和 AR Time。
7. PM 在“项目列表”或项目详情页查看成本、进度、绩效、AR Time 和人员预算 vs AR 明细。

## 三、Pre-cal 使用说明

### 进入 Pre-cal 页面

Sales 用户从小程序底部进入“业务”页，在“售前 Pre-cal”区域可以看到：

- “新建 Pre-cal”：创建新的 AUD Pre-cal。
- “我的 Pre-cal”：查看自己创建的 Pre-cal，包括草稿、已提交、已撤销、已绑定 SAP 等状态。

### 新建 Pre-cal

操作步骤：

1. 点击“业务”页中的“新建 Pre-cal”。
2. 填写基础信息：
   - `Customer Name`：客户名称。
   - `Service`：服务类型，目前界面提供 `ESG`、`CSR`。
   - `Remark`：备注，可填写项目背景或特殊说明。
3. 在“服务明细”中填写一条或多条明细：
   - `Service/Product Description`：服务或产品描述。
   - `Order Create Center`：订单创建中心。
   - `Order Value`：该明细对应的订单金额。
   - `Onsite MD`：现场服务人天。
   - `Offsite MD`：非现场服务人天。
   - `Quotation MD`：报价阶段人天。默认按系统规则计算，也可以切换为手动填写。
   - `Travel MD`：差旅人天。默认按系统规则计算，也可以切换为手动填写。
   - `翻译费`、`设计费`、`差旅费`、`Other ext.`、`Subcontracting IC`、`Internal subcon`、`Other project related costs`：项目相关外部或内部成本。
4. 如有多个服务明细，点击“新增服务明细”；不需要的明细可点击“删除”。
5. 查看“计算结果”和“70% / 80% Productivity 情景”。
6. 点击“保存”保存草稿，或点击“保存并提交”提交给 CS 处理。

### 字段说明

- `Order Value`：订单金额，是 Pre-cal 预算和利润测算的重要输入。
- `Total MD`：系统根据 Onsite MD、Offsite MD、Quotation MD、Travel MD 汇总出的总人天。
- `Hours`：Total MD 换算出的工时。
- `差旅费`：服务明细中的差旅相关成本，后续创建项目时会带入项目差旅费或子项目差旅费。
- `Operating Margin`：系统根据订单金额、成本、人天和费用计算出的经营利润率，用于评估项目报价质量。
- `precalNo`：如果页面展示 Pre-cal 编号，它只是系统内部编号或辅助识别号，不是 SAP 项目号，也不是正式项目号。

注意事项：

- Pre-cal 创建后，不能直接用于项目创建；需要先由 CS 绑定 SAP 项目号。
- 如果只点击“保存”，记录仍可能是草稿，CS 页面不会处理草稿记录。
- 如果需要让 CS 继续绑定 SAP，请使用“保存并提交”或在“我的 Pre-cal”里提交。

## 四、SAP 绑定说明

SAP 项目号是和 Pre-cal 绑定的。PM 后续创建项目时，系统会根据输入的 SAP 项目号反查对应 Pre-cal。

### 进入 SAP 绑定页面

CS 用户从小程序底部进入“业务”页，在“CS 处理”区域点击“SAP 绑定”。

SAP 绑定列表会显示 `Submitted`、`SAP Bound`、`Project Created` 状态的 Pre-cal；草稿和已撤销记录不会出现在 CS 处理列表中。

### 绑定 SAP

操作步骤：

1. 进入“SAP 绑定”列表。
2. 可按客户、Sales 或 SAP 号搜索 Pre-cal。
3. 点击需要处理的 Pre-cal。
4. 在“SAP 绑定信息”中填写：
   - `SAP号`：正式 SAP 项目号。
   - `Item号`：对应 Item 号；界面提示 7 开头 SAP 默认 1000。
   - `关联人员`：可选，用于说明该 SAP 与哪位人员或填报关系相关。
   - `备注`：可选。
5. 如一个 Pre-cal 对应多个 SAP，点击“+ 新增 SAP号”继续添加。
6. 在“Item 信息”中维护 Item 号、Item 说明和备注。
7. 填写“保存原因 / 备注”，点击“保存 SAP 绑定”。

### 业务说明

- 一个 Pre-cal 可以绑定多个 SAP 项目号。
- 多个 SAP 项目号可能来自不同组员、不同服务拆分或不同 AR 填报关系。
- 绑定 SAP 后，创建项目时输入任意一个已绑定的 SAP 项目号，都应能找到对应的 Pre-cal。
- 如果后期需要删除或恢复 SAP 绑定，请按页面提示操作，并在保存原因中说明修改原因。
- SAP 绑定会影响后续项目创建和 AR Time 匹配，请确认 SAP 号输入准确，避免多输空格或填错数字。

## 五、创建项目说明

### 进入创建项目页面

PM / 项目负责人从小程序底部进入“业务”页，在“项目管理”区域点击“创建项目”。

### 通过 SAP 创建项目

推荐操作步骤：

1. 在“通过 SAP 项目号创建（推荐）”输入框中输入任意一个已绑定 SAP 项目号。
2. 点击“同步 Pre-cal 数据”。
3. 如果系统找到对应 Pre-cal，会显示“已同步 Pre-cal 信息”。
4. 确认页面自动带入的项目数据。
5. 根据实际情况补充项目名称、开始日期、结束日期、项目组员、人员预算工时、计划完成工时等信息。
6. 点击“保存项目”。

成功匹配后，系统会尽量自动带入：

- 客户名称 / 合同主体
- 项目号 / 主 SAP 项目号
- 项目预算 / BAC 相关数据
- 可分配工作工时 / 预算工时
- 差旅费
- 子项目列表
- 成员信息
- SAP / Item 信息
- Pre-cal 的 Service、Sales Owner、Operating Margin 等辅助信息

### 找不到 SAP 项目号时

如果点击“同步 Pre-cal 数据”后提示找不到对应 Pre-cal，请检查：

- 该 SAP 是否已经由 CS 绑定到 Pre-cal。
- SAP 号是否输入正确。
- 是否多输了空格、换行或其他字符。
- Pre-cal 是否已经提交，而不是仍停留在草稿状态。
- 是否存在多个 Pre-cal 绑定了同一个 SAP，需要先确认绑定关系。

## 六、项目列表与项目详情说明

### 项目列表

从底部“项目”页进入项目列表。PM 默认查看自己创建的项目。

项目列表支持：

- 搜索项目名称、项目号、客户或 PM。
- 按“全部 / 风险 / 正常”筛选。
- 点击项目卡片进入编辑或查看页面。

项目列表中可以看到：

- 项目名称
- 项目号
- 客户名称
- PM
- 风险 / 正常状态
- BAC（不含差旅）
- 项目总预算（含差旅）
- 人员预算工时合计
- AC（实际成本）
- CPI（成本绩效）
- SPI（进度绩效）
- 实际完成率
- 计划完成率
- 异常提醒

### 项目详情

项目详情页可以查看和维护：

- 项目基本信息：项目名称、客户名称、项目号、日期、PM、项目组员、项目状态。
- SAP 绑定：当前有效 SAP、Item、关联人员、备注，以及历史 SAP。
- 子项目预算与计划：子项目名称、Item No.、预算工时、子项目差旅费、报价人天单价、计划完成工时。
- 人员预算工时分配：可分配工时、已分配工时、剩余工时、分配比例、每个人的预算工时。
- 成本参数：差旅费、内部人天成本。
- AR Time：AR Time 总计、匹配汇总数、AR 更新时间、人员 / SAP / Item 明细。
- 自动计算预览：BAC、PV、EV、AC、CV、SV、CPI、SPI、人员预算 vs AR 明细和异常提醒。

注意事项：

- AR Time 来自 AR Summary，小程序页面只展示同步结果，不写回 Excel。
- 项目里的绩效指标由系统自动计算，普通用户主要维护基础信息、预算工时、计划完成工时和成员分配。

## 七、AR Time 匹配逻辑说明

AR Time 来源于 AR Summary 数据。AR Summary 通常由 AR 数据上传或同步后生成，项目页面会根据 SAP 项目号匹配 AR 数据。

业务规则：

- 系统按照 SAP 项目号匹配 AR Summary。
- AR Time 不要求 Item No. 强匹配；Item No. 主要用于明细展示和分组。
- 如果一个 Pre-cal 绑定了多个 SAP 项目号，创建项目时输入其中任意一个 SAP，系统应关联该 Pre-cal 下相关 SAP 的 AR Time。
- AR Summary 每次上传后，如果数据有更新，项目页面会以系统当前可查询到的最新结果为准。
- 没有出现在本次上传 Summary 中的旧记录，不应简单理解为工时变成 0；需要结合系统当前的同步和统计逻辑判断是否参与统计。
- 项目页面上的 AR Time 可能会随着 AR Summary 更新而变化。
- 如果页面显示异常，用户可先检查 SAP 绑定是否正确，以及 AR Summary 中是否包含对应 SAP 项目号。

项目详情页 AR Time 区域会显示：

- `AR Time 总计`：匹配到的 AR 工时合计。
- `匹配汇总数`：匹配到的 AR Summary 汇总记录数量。
- `AR 更新时间`：当前匹配结果中最新的 AR 更新时间。
- 明细：员工姓名、SAP、Item、AR Time、Record Count。

## 八、成本、进度、绩效字段说明

以下字段由系统根据项目输入、Pre-cal 数据和 AR Time 自动计算或展示。

### 预算与成本

- `BAC（不含差旅）`：根据子项目预算工时和报价人天单价计算，是系统用于 PV / EV 绩效计算的预算基准。
- `杂费 / 差旅费`：项目或子项目中的差旅相关费用。
- `项目总预算（含差旅）`：BAC 加上差旅费，仅作为项目预算展示值，不参与 PV / EV 计算。
- `预算工时`：主要来自 Pre-cal 的工作人天换算，创建项目后会体现在子项目预算工时和人员预算分配中。
- `人员预算工时合计`：项目人员分配的预算工时总和。
- `实际 AR Time`：来自 AR Summary 的实际填报工时。
- `AC（实际成本）`：系统根据 AR 工时合计、8 小时 / 人天和内部人天成本换算。

### 进度与绩效

- `计划完成率`：根据“截至当前复盘节点计划完成工时”和子项目预算工时计算。
- `PV（计划价值）`：按计划完成率折算出的计划价值。
- `实际完成率`：根据 AR 工时和子项目预算工时计算。
- `EV（挣值）`：按实际完成率折算出的挣值。
- `CV（成本偏差）`：EV 与 AC 的差异。为负时，通常表示存在超支或亏损风险。
- `SV（进度偏差）`：EV 与 PV 的差异。为负时，通常表示实际进度落后于计划。
- `CPI（成本绩效）`：成本效率指标，小于 1 时表示成本效率低于预期。
- `SPI（进度绩效）`：进度效率指标，小于 1 时表示进度低于计划。

注意事项：

- 如果页面显示 `-` 或空值，通常表示缺少必要输入，例如预算工时、计划完成工时或 AR Time。
- 这些指标是展示和预览值，不建议把系统自动带入或自动计算的预算基准随意手动改成与 Pre-cal 不一致。

## 九、常见问题 FAQ

### 1. 输入 SAP 项目号后找不到 Pre-cal 怎么办？

先确认 SAP 已经在 CS 的“SAP 绑定”页面绑定到对应 Pre-cal，并且 SAP 号输入正确。注意不要多输入空格、换行或错误数字。如果 Pre-cal 仍是草稿，也需要先提交后才能进入后续绑定流程。

### 2. 为什么项目里没有 AR Time？

常见原因包括：项目没有有效 SAP 绑定、AR Summary 里没有对应 SAP、AR 数据还没有完成上传或同步、输入的 SAP 与 AR Summary 中的 SAP 不一致。

### 3. 为什么 AR Time 和 Excel / AR Summary 里看到的不一致？

项目页展示的是系统当前匹配到的 AR Summary 汇总结果。请检查 AR Summary 是否已经完成最新上传，SAP 号是否一致，以及项目是否关联了正确的 Pre-cal 和 SAP 列表。

### 4. 为什么一个 Pre-cal 可以有多个 SAP？

同一个 Pre-cal 可能拆分成多个 SAP 项目号，也可能因为不同组员、不同服务内容或不同 AR 填报关系需要多个 SAP。系统允许一个 Pre-cal 绑定多个 SAP，创建项目时输入任意一个已绑定 SAP 都应能找到该 Pre-cal。

### 5. SAP 绑定后还能不能修改？

可以在业务页面按提示删除、恢复或新增 SAP，并填写保存原因。SAP 修改会影响后续项目创建和 AR Time 匹配，操作前请确认影响。

### 6. 为什么有些数字是空的，不显示 0？

空值通常表示系统缺少必要输入，无法计算该指标；0 则表示该字段有明确的数值结果。为了避免误解，系统会保留部分空值显示。

### 7. 为什么项目预算 / BAC 不建议手动改？

BAC 是后续成本、进度和绩效计算的基础，通常应来自 Pre-cal 和项目预算数据。随意修改可能导致 CPI、SPI、CV、SV 等指标失真。

### 8. 成员信息是从哪里来的？

项目创建者会作为 PM。项目组员由 PM 在项目创建或编辑页输入；点击“同步 PM/组员”后，可把 PM 和组员同步到人员预算工时分配。AR 人员来自 AR Summary，只作为实际工时展示。

### 9. 子项目编号 1000 / 2000 / 3000 是什么意思？

这些是 Item No. 或子项目编号。通过 Pre-cal 导入时，服务明细会生成子项目，常见 Item No. 会按顺序使用 1000、2000、3000。它们用于区分子项目和明细展示；AR Time 主要按 SAP 项目号匹配。

### 10. 保存后页面没有变化应该怎么处理？

可以返回项目列表重新进入，或在页面下拉刷新。若仍未变化，请确认保存时是否有错误提示、网络是否正常，以及当前项目是否为只读查看状态。

## 十、使用提醒

- 首次使用时，如果页面提示姓名缺失，请到“我的”页填写姓名。
- Sales 负责创建和提交 Pre-cal。
- CS 负责在业务页面维护 SAP 绑定。
- PM / 项目负责人负责通过 SAP 创建项目、维护项目预算和查看项目执行情况。
- 本 README 面向普通业务用户，只说明日常业务操作。
